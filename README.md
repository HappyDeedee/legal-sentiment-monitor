# legal-sentiment-monitor

律所舆情运营系统，用于帮助运营人员配置监控任务、采集公开平台搜索结果、做疑似负面线索初筛，并生成报告和邮件。

系统第一版聚焦“可配置、可运行、可查看、可追踪”的闭环：运营人员在后台维护律所、关键词、平台账号、代理资源、AI 接入、AI 评估规则和邮件模板；系统按频率执行采集，完成去重、时间过滤、AI 初筛、报告生成和邮件发送。

> AI 结果只作为“疑似负面线索筛查”，不代表事实认定。采集和使用公开信息时，应遵守目标平台规则、法律法规和项目许可证约束。

## 当前能力

- 总览：展示任务、运行、内容、风险线索、报告、账号、代理和 AI 接入概览。
- 舆情监控：创建任务、编辑任务、立即运行、暂停、删除。
- 运行中心：查看运行状态、运行日志、失败原因和停止入口。
- 报告中心：预览 HTML 报告，下载 Excel、Markdown、HTML，筛选线索明细。
- 资源管理：维护平台账号、代理资源和 AI 接入，为后续绑定任务预留。
- 系统配置：维护 AI 评估规则、SMTP 邮件配置和 HTML 邮件模板。
- 平台范围：第一版优先跑通抖音闭环，快手和小红书保留配置与扩展入口。
- 数据存储：SQLite 和本地报告文件，后续可迁移到 PostgreSQL。

## 快速启动

在项目根目录启动后台：

```powershell
.\start_webui.bat
```

打开管理后台：

```text
http://127.0.0.1:8080/monitor
```

服务器模式可使用：

```powershell
.\start_monitor_service.bat
```

## 后台功能

### 总览

总览页只展示运营状态：任务总数、启用任务、今日运行、今日采集内容、疑似负面、高风险、待复核、报告数、账号资源、代理资源、AI 接入数量、平台登录状态、调度器状态、最近失败任务和最近报告。

### 舆情监控

基础监控配置包括：

- 律所名称、律所别名、关键词、排除词
- 平台：抖音、快手、小红书
- 是否采集评论
- 采集范围：最近 1 天、7 天、30 天、自定义日期
- 抓取频率：每天、每 12 小时、每 6 小时、自定义 cron
- 收件邮箱
- 绑定 AI 接入
- 绑定邮件模板
- 可选绑定账号资源、代理资源
- 启用或暂停

高级采集配置默认折叠，包含：

- 采集目标类型：关键词搜索、指定内容链接、指定用户主页
- 最大采集页数、最大采集条数、起始页
- 是否采集二级评论
- 输出方式：系统内部存储、JSON、Excel
- 浏览器模式：本地窗口、服务器扫码登录、已有登录态
- 平台差异能力说明

后台页面使用业务语言展示配置，不直接暴露底层采集参数。

### 运行中心

运行中心展示运行 ID、任务 ID、任务名称、平台、关键词摘要、状态、开始时间、耗时、采集数、新增数、疑似负面数、高风险数、待复核数、失败原因和操作。

日志以悬浮窗展示，默认定位到最新内容，并支持刷新、复制、下载。即使任务被删除，历史运行仍会保留；如果对应进程仍在运行，也可以从运行中心停止。

### 报告中心

报告中心支持按律所、平台、风险等级和日期筛选。点击不同报告时，HTML 预览和线索明细会同步切换。

默认第一版：

- 内容链接跳转平台原始链接。
- 封面展示平台原始封面链接。
- 不默认保存封面图片。

可选开启“封面归档”：系统会尝试把封面保存到本地资源目录。保存失败不影响报告生成；有本地封面时优先展示本地封面，否则展示原始封面链接。

报告统一使用“疑似负面线索”表述，并保留声明：AI 仅作线索筛查，不代表事实认定。

### 账号资源

账号池第一版只做资源管理、登录状态维护和任务绑定预留，不做自动轮换、不做复杂号池调度、不处理验证码绕过。

字段包括：账号名称、平台、登录方式、状态、绑定代理、网页登录态标识、备注、错误信息、最近检测时间。

登录方式包括：

- 二维码：页面发起登录，展示二维码、状态和轮询结果。
- Cookie：保存 Cookie，页面只显示掩码。
- 已有网页登录态：保存登录态标识。
- 本地窗口登录：作为本地开发或远程桌面场景的兜底入口。

如果遇到滑块、短信、验证码或二维码获取失败，系统只把状态、截图或提示返回前端，不做绕过。

### 代理资源

代理池第一版只做资源管理和绑定预留，不做自动换 IP、不做代理质量评分、不做动态代理调度。

字段包括：代理名称、供应商、代理 URL、状态、最大并发、备注、错误信息、最近检测时间。代理 URL 加密保存，前端只显示掩码。

### AI 接入和评估规则

资源管理中的 AI 接入支持多个模型连接资源，每个接入包含 Provider、Base URL、API Key、Model、Temperature、启用状态和默认状态。API Key 加密保存，前端只显示掩码。

系统配置中的 AI 评估规则只维护相关性、疑似负面和风险等级 Prompt。AI 未配置时不影响采集，内容会进入待人工复核；AI 连接测试只发送简单消息确认接口可用。

### 邮件配置和模板

保留 SMTP 配置，并支持多个 HTML 邮件模板。模板包含标题模板和 HTML 内容，编辑时支持实时预览。

支持变量：

- 律所名称
- 日期
- 新增内容数
- 疑似负面数
- 高风险数
- 待复核数
- 平台
- 报告正文

报告发送时优先使用当前启用模板。

## 平台登录态

服务器运行前，需要为目标平台准备登录态。第一版推荐在后台“账号资源”里维护账号：

- 抖音：二维码、手机号、Cookie、已有网页登录态。
- 快手：二维码、Cookie、已有网页登录态；手机号登录入口暂不开放。
- 小红书：二维码、手机号、Cookie、已有网页登录态。

服务器场景下，运营人员不依赖服务器本地窗口。后台会创建登录会话，前端展示二维码、登录状态或需要人工处理的验证提示；扫码成功后系统保存网页登录态，后续定时任务复用该登录态。

“本地窗口登录”只作为兜底方式，适合本地开发、远程桌面或二维码回传不稳定时使用。

## 数据和安全

- AI Key、SMTP 密码、Cookie、代理 URL 加密保存。
- 前端只展示掩码。
- 运行日志和错误提示会尽量脱敏。
- 平台网页登录态不应提交到 Git 仓库。
- 建议定期备份 SQLite、报告目录、加密密钥和平台网页登录态。

## 交付边界

第一版只做内部运营可用的轻量业务系统，不做：

- 多租户
- 复杂账号轮换
- 高并发采集平台
- 平台级反验证能力
- 商业化权限系统

后续可逐步增强账号池调度、代理调度、PostgreSQL、多 Worker、权限管理和部署自动化。
