# legal-sentiment-monitor 操作说明

本文面向运营和交付人员，说明第一版律所舆情运营系统的使用方式。

## 启动后台

在项目根目录启动：

```powershell
.\start_webui.bat
```

后台地址：

```text
http://127.0.0.1:8080/monitor
```

服务器运行可使用：

```powershell
.\start_monitor_service.bat
```

第一版调度器运行在单个后台进程内。部署时不要同时开多个后台进程；如果必须由外部定时器调度，请关闭内置调度器，并使用到期任务执行入口。

## 数据目录

默认数据目录：

```text
monitor_data/
browser_data/
```

建议服务器显式指定：

```powershell
$env:MONITOR_DATA_DIR = "D:\legal-sentiment-monitor\data"
$env:MONITOR_BROWSER_DATA_DIR = "D:\legal-sentiment-monitor\browser_data"
```

- `MONITOR_DATA_DIR`：保存数据库、加密密钥、运行日志和报告文件。
- `MONITOR_BROWSER_DATA_DIR`：保存平台网页登录态目录。

系统会对常见密钥、密码、Cookie、Token、代理 URL 等敏感内容做掩码展示。实际使用时，不要把密钥、Cookie 或验证码写入律所名称、关键词、Prompt 等业务字段。

## 菜单结构

后台按运营工作流组织：

- 总览：查看系统运行状态、最近失败、最近报告和常用入口。
- 舆情监控：维护任务列表、新建任务、查看任务详情。
- 运行中心：查看运行记录、运行状态、失败原因和实时日志。
- 报告中心：预览报告、筛选线索、下载附件、查看发送记录。
- 资源管理：维护平台账号、代理资源和 AI 接入。
- 系统配置：维护 AI 评估规则、邮件配置、邮件模板和系统诊断。

## 舆情监控任务

基础监控配置：

- 律所名称
- 律所别名
- 关键词
- 排除词
- 平台：抖音、快手、小红书
- 是否采集评论
- 采集范围：最近 1 天、7 天、30 天、自定义日期
- 抓取频率：每天、每 12 小时、每 6 小时、自定义 cron
- 收件邮箱
- 绑定 AI 接入
- 绑定邮件模板
- 可选绑定账号资源
- 可选绑定代理资源
- 启用或暂停

高级采集配置默认折叠：

- 采集目标类型：关键词搜索、指定内容链接、指定用户主页
- 最大采集页数、最大采集条数、起始页
- 是否采集二级评论
- 输出方式：系统内部存储、JSON、Excel
- 浏览器模式：本地窗口、服务器扫码登录、已有登录态
- 平台差异能力说明

“邮件发送时间”同时作为定时采集的当天触发锚点：每日任务会在该时间运行，6 小时 / 12 小时任务会以该时间作为当天起点，报告在本轮采集、去重和 AI 评估完成后发送。

点击“立即运行”前，系统会检查任务是否正在运行、关键词和平台是否完整、平台登录材料是否可用、账号绑定代理是否可用、AI 和邮件配置是否具备。硬性问题会阻止启动；非硬性问题会提示运营确认后继续。

## 运行中心

运行中心展示：

- 运行 ID
- 任务 ID
- 任务名称 / 律所
- 平台
- 关键词摘要
- 状态
- 开始时间
- 耗时
- 采集数
- 新增数
- 疑似负面数
- 高风险数
- 待复核数
- 失败原因
- 操作

关键词过长时会省略展示，日志使用悬浮窗展示并默认定位到最新内容。日志支持刷新、复制和下载。

如果任务已经删除，历史运行仍会显示“原任务已删除”；如果进程仍在运行，可以继续从运行中心停止。

## 报告中心

报告中心支持按律所、平台、风险等级和日期筛选。点击不同报告时，HTML 预览和线索明细会同步切换。

第一版链接和封面逻辑：

- 内容链接跳转平台原始链接。
- 封面默认使用平台原始封面链接。
- 不默认保存图片。

可选能力：

- 开启“封面归档”后，系统尝试保存封面到本地资源目录。
- 下载失败不影响报告生成。
- 有本地封面时优先展示本地封面，否则展示原始封面链接。

报告统一使用“疑似负面线索”表述。底部声明：AI 仅作线索筛查，不代表事实认定。

## AI 接入和评估规则

资源管理支持多个 AI 接入资源：

- Provider：OpenAI Compatible / Anthropic
- Base URL
- API Key
- Model
- Temperature
- 启用状态
- 默认状态

API Key 加密保存，前端只显示掩码。AI 连接测试只发送最简单消息确认接口、鉴权和模型可用。

系统配置中的 AI 评估规则维护相关性、疑似负面和风险等级 Prompt。AI 未配置或调用失败不影响采集和报告生成，内容会进入“待人工复核”。

AI 输出应为固定 JSON 字段：

- `is_related`
- `is_negative`
- `risk_level`
- `reason`
- `evidence_quotes`
- `recommended_action`

AI 调用失败或 JSON 解析失败时，内容标记为“待人工复核”，不阻断报告生成。

## 邮件配置和模板

邮件配置包括 SMTP Host、Port、SSL/STARTTLS、发件人、用户名、密码、默认收件人和标题模板。SMTP 密码加密保存，前端只显示掩码。

邮件模板支持标题模板和 HTML 内容，编辑时实时预览。

模板变量：

- 律所名称
- 日期
- 新增内容数
- 疑似负面数
- 高风险数
- 待复核数
- 平台
- 报告正文

报告发送时优先使用当前启用模板；没有风险线索时也发送正常日报。

## 平台账号

账号池第一版只做资源管理、登录状态维护和任务绑定预留，不做自动轮换、不做复杂调度、不绕过验证码。

字段：

- 账号名称
- 平台
- 登录方式
- 状态
- 绑定代理
- 登录态目录
- 备注
- 错误信息
- 最近检测时间

登录方式：

- 二维码：前端展示发起登录、二维码和状态轮询。
- Cookie：展示 Cookie 输入框，保存后只显示掩码。
- 已有网页登录态：展示登录态目录输入。
- 本地窗口登录：展示打开本地登录窗口按钮，作为兜底方式。

服务器部署时，推荐使用“页面发起登录”：系统创建登录会话，前端展示二维码或验证状态，运营人员扫码或按平台提示处理，系统轮询登录结果并保存网页登录态。

如果遇到滑块、短信、验证码或二维码获取失败，系统会把状态、截图或提示返回前端，不做绕过。

## 代理资源

代理池第一版只做资源管理和绑定预留。

字段：

- 代理名称
- 供应商
- 代理 URL
- 状态：可用、停用、异常
- 最大并发
- 备注
- 错误信息
- 最近检测时间

代理 URL 加密保存，前端只显示掩码。第一版不做自动换 IP、不做代理评分、不做动态调度。

## 数据处理规则

- 排除词在入库前生效：标题、正文、关键词、作者命中任一排除词时，不进入 AI 评估和报告。
- AI 判断必须同时满足“与目标律所相关”和“疑似负面”，才计入疑似负面线索。
- 去重按“任务 + 平台 + 内容 ID”生效，同一任务不会重复发送同一条内容。
- 不同律所任务可分别评估同一条内容，避免多家律所监控互相影响。
- 指定最近 1 天时，时间范围外内容不进入报告。

## 系统诊断

系统诊断用于检查项目文件、数据目录、数据库、平台登录配置、AI 配置、邮件配置、任务、报告链路和调度环境。诊断不会调用真实平台、AI 或 SMTP。

测试数据统一使用：

- 律所：海安律所
- 关键词：海安律所避雷、海安律所退费、海安律所投诉

测试环境不调用真实 AI 服务。
